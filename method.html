Position sizing: Current arena rule invests up to 30% of available cash per buy signal

Quant AI — Four-Vote Composite System

A systematic trading strategy I designed from scratch that combines three rule-based technical signals with a machine learning model. A weighted voting mechanism aggregates four independent sub-signals into a final trading decision. The system runs daily via automated cron, downloading fresh market data, generating signals, executing simulated trades, and persisting results to SQLite.

Four Sub-Signals

1. MA Crossover

  • Logic: MA20 crosses above/below MA50
  • Signal: MA20 > MA50 → bullish (+1), else bearish (-1)
  • Type: Trend-following

2. RSI Mean Reversion

  • Logic: RSI(14) oversold/overbought thresholds
  • Signal: RSI < 30 → bullish, RSI > 70 → bearish
  • Type: Mean reversion

3. Bollinger Band

  • Logic: Price breaks 2σ bands around MA20
  • Signal: Price < lower band → bullish, > upper → bearish
  • Type: Volatility breakout

4. ML Prediction

  • Model: Random Forest Classifier
  • Target: 5-day forward return direction (binary)
  • Type: Predictive classification

Weighted Voting & Position Sizing

3-4 bullish votesBuy 50% of cash
2 bullish votesBuy 25% of cash
2 bearish votesSell 50% of position
3-4 bearish votesSell 100% of position
Mixed / neutralHold

Feature Engineering (24 Features)

Price Features (per window: 5, 10, 20, 60d)

  • return_Nd — N-day cumulative return
  • volatility_Nd — rolling std of daily returns
  • ma_dist_Nd — (price - MA) / MA, distance from mean
  • high_low_ratio_Nd — rolling max(High) / min(Low)
  • volume_change_Nd — N-day volume change rate

= 20 features (5 metrics × 4 windows)

Technical Indicators (4 features)

  • rsi_14 — Relative Strength Index (14-period)
  • macd — EMA(12) - EMA(26)
  • macd_signal — EMA(9) of MACD line
  • bb_position — (price - MA20) / (2 × std20), position within Bollinger Bands
  • atr_ratio — ATR(14) / Close, normalized volatility

Note: macd_hist (MACD - signal) also computed

Validation & Anti-Leakage

Validation methodWalk-Forward (TimeSeriesSplit, 5 folds, expanding window)
Data leakage preventionChronological split — train on past, test on future only
Look-ahead bias preventionNext-bar execution: signals use data up to day T, trades execute at day T+1 open
Target constructionForward return = Close[T+5] / Close[T] - 1 (shifted backward, no peeking)
MetricsAccuracy, Precision, Recall, F1 (per fold + mean)
Test coverage81% (44 unit tests passing)

Retrain & Refresh Pipeline

# Daily cron (after US market close, ~14:00 PDT)
python3 daily_runner.py --end $(date +%F)

# 1. Download fresh OHLCV from yfinance (SPY, QQQ, VOO, VGT, SMH)
# - 420-day warmup for indicator calculation
# 2. Build features (24 per ticker per day)
# 3. Run 4 sub-signal generators
# 4. Vote → position sizing → simulated trade execution
# 5. Record signals, equity, holdings to SQLite

# Weekly model retrain
python3 retrain.py --end $(date +%F) --splits 5

# 1. Download ~900 days of history per ticker
# 2. Prepare full feature/target dataset (leakage-checked)
# 3. Walk-forward validate (5 expanding-window folds)
# 4. Fit final model on ALL available data
# 5. Save .joblib per ticker + training_metrics.json

Deterministic backfill: The entire pipeline is designed for full reset and rebuild. clear_source_data() wipes all Quant AI rows from SQLite, then simulate_portfolio() replays from the starting date (Jun 15, 2026) using the same signal logic. This ensures reproducibility — running the same data through the pipeline always produces identical results.
